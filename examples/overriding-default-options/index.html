Overriding default options

← Examples

{% include share.html %}

Assume that your page consists of many forms that need to be validated. These forms might use same options, such as icon, default error message, etc.

As a natural approach, you have to repeat initializing the options as following:

{% highlight javascript %} $(document).ready(function() { $(firstForm).formValidation({ message: 'The field is not valid', icon: { valid: 'glyphicon glyphicon-ok', invalid: 'glyphicon glyphicon-remove', validating: 'glyphicon glyphicon-refresh' } ... }); $(secondForm).formValidation({ message: 'The field is not valid', icon: { valid: 'glyphicon glyphicon-ok', invalid: 'glyphicon glyphicon-remove', validating: 'glyphicon glyphicon-refresh' } ... }); }); {% endhighlight %}

You can override the default options as below:

{% highlight javascript %} $(document).ready(function() { $.fn.formValidation.DEFAULT_OPTIONS = $.extend({}, $.fn.formValidation.DEFAULT_OPTIONS, { message: 'The field is not valid', icon: { valid: 'glyphicon glyphicon-ok', invalid: 'glyphicon glyphicon-remove', validating: 'glyphicon glyphicon-refresh' } }); }); {% endhighlight %}

By overriding the default options, you don't need to repeat setting these options when calling the plugin:

{% highlight javascript %} $(document).ready(function() { $(firstForm).formValidation({ // Don't need to set the icon, message options anymore ... }); $(secondForm).formValidation({ // Don't need to set the icon, message options anymore ... }); }); {% endhighlight %} {% include demo/tabs.html id="demo" frameworks="bootstrap" codes="programmatic" %}
