Settings structure

There are three levels of settings which are form, field, and validators.

The BootstrapValidator plugin can be called as following:

{% highlight javascript %} $(formSelector).bootstrapValidator({ ... Form settings go here ... fields: { specificFieldName: { ... Field settings go here ... validators: { specificValidatorName: { ... common validator settings go here ... ... specific validator settings ... } } } } }); {% endhighlight %}

Most of settings can be set via HTML 5 attributes prefixed with data-bv.

For example, the following call:

{% highlight javascript %} $(formSelector).bootstrapValidator({ excluded: [':disabled', ':hidden', ':not(:visible)'], feedbackIcons: { valid: 'glyphicon glyphicon-ok', invalid: 'glyphicon glyphicon-remove', validating: 'glyphicon glyphicon-refresh' }, live: 'enabled', message: 'This value is not valid', submitButtons: 'button[type="submit"]', trigger: null, fields: ... }); {% endhighlight %}

is equivalent with the one below:

{% highlight html %}
...
{% endhighlight %} {% highlight javascript %} $(document).ready(function() { $(formSelector).bootstrapValidator({ fields: ... }); }); {% endhighlight %}
