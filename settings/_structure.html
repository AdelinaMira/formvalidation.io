Settings structure

There are three levels of settings which are form, field, and validators.

The FormValidation plugin can be called as following:

{% highlight javascript %} $(formSelector).formValidation({ // Indicate the framework // It can be: bootstrap, foundation, pure, semantic, uikit framework: 'bootstrap', // ... Form settings go here ... fields: { specificFieldName: { // ... Field settings go here ... validators: { specificValidatorName: { // ... common validator settings go here ... // ... specific validator settings ... } } } } }); {% endhighlight %}

Most of settings can be set via HTML 5 attributes prefixed with data-bv.

For example, the following call:

{% highlight javascript %} $(formSelector).formValidation({ framework: 'bootstrap', excluded: [':disabled', ':hidden', ':not(:visible)'], icons: { valid: 'glyphicon glyphicon-ok', invalid: 'glyphicon glyphicon-remove', validating: 'glyphicon glyphicon-refresh' }, live: 'enabled', message: 'This value is not valid', trigger: null, fields: ... }); {% endhighlight %}

is equivalent with the one below:

{% highlight html %}
...
{% endhighlight %} {% highlight javascript %} $(document).ready(function() { $(formSelector).formValidation({ fields: ... }); }); {% endhighlight %}
